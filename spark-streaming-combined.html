Two Modes of Spark Streaming

Microbatch Mode

  • Traditional approach
  • 1-2 second latency
  • More cost effective
  • Great for most use cases
  • Use trigger(processingTime="1 second")

Real-Time Mode (RTM)

  • Sub-second latency
  • P99 latency as low as 5ms
  • Workers pre-allocated
  • No scheduling wait
  • Use trigger(realTime="1 minutes")

DBR 16.4 LTS+ · Public Preview

It's Just the Flip of a Switch

Your code doesn't change. Only the trigger configuration:

# Microbatch Mode (1-2 second latency)
.trigger(processingTime="1 second")

# Real-Time Mode (sub-second latency)
.trigger(realTime="1 minutes")

Write your application normally. Decide at the end whether you need real-time mode or not.

When Kafka Still Wins

Exactly-Once Semantics

Financial transactions, order processing. Zerobus is at-least-once (exactly-once on roadmap).

Fan-Out Patterns

Multiple consumers reading the same stream with different processing logic.

Message Retention & Replay

Need to replay messages? Kafka's retention model is essential.

Zerobus is for direct lakehouse writes. Multi-consumer or replay? Use Kafka.
